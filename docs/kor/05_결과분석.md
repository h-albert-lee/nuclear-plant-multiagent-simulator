# 결과 분석 — `runs/{id}/` 의 raw 데이터 + report 읽기

## 출력 디렉토리 구조

run 종료 후 `runs/{run_id}/` 에 다음이 작성됩니다:

```
runs/{run_id}/
├── manifest.json                          # 아티팩트 목록 + sha256 + line_count
├── config.json                            # 사용된 전체 config snapshot
├── messages.jsonl                         # 모든 bus 메시지 (canonical)
├── guardrail.jsonl                        # 가드레일 판정 trace
├── plant_state.jsonl                      # 매 tick plant 스냅샷
├── ingress.jsonl                          # 외부 ingress 입력 trace
├── approvals.jsonl                        # mock_human 응답
├── attack_sessions.jsonl                  # 어택 세션 lifecycle + situation_summary
├── agent_thoughts.jsonl                   # LLM 호출 trace (prompt sha256 + output)
├── agent_thoughts_prompts/                #   dedup 된 prompt 본문
│   └── {sha256}.txt
├── safety_function_timeline.jsonl         # CSF 상태 전환
├── run_summary.json                       # 종료 헤더 (사유, 최종 CSF)
├── report.json                            # 구조화 분석 report (9 섹션)
└── report.md                              # 사람 가독 markdown
```

## 빠른 검토

```bash
ID=$(ls -t runs/ | head -1)
cat runs/$ID/report.md            # 사람 가독 한눈에 보기
```

`report.md` §1 요약을 먼저 확인:
- 안전기능 손상 개수 (0/6 이 정상)
- 권한위반 건수 (낮을수록 안전)
- 어택 세션 수

## report.json 의 9 섹션

```python
import json
r = json.load(open(f"runs/{ID}/report.json"))

r["metadata"]                  # 시작/종료 tick, 종료 사유
r["configuration_summary"]     # 어떤 가드레일·STA mode 로 돌았는지
r["attack_summary"]            # 세션별 / 채널별 분포
r["guardrail_summary"]         # G_i 별 invocation·block 카운트
r["plant_trajectory"]          # op_state 전환 + CSF timeline + alarm 카운트
r["agent_message_stats"]       # sender × type × action_class × action_type
r["violations"]                # 권한 위반 / bypass 시도 / 분류 오류 / 보고 누락
r["notable_events"]            # SCRAM / EOP 진입 / CSF lost 등
r["reproducibility"]           # config·scenario·seed sha
```

## raw JSONL 직접 파싱

JSONL 은 한 줄 = 하나의 JSON 객체. Python:

```python
import json
from pathlib import Path

def read_jsonl(path):
    with open(path, "rt", encoding="utf-8") as f:
        for line in f:
            yield json.loads(line)

# 예: 모든 ACTION 메시지 추출
for ev in read_jsonl(f"runs/{ID}/messages.jsonl"):
    msg = ev["msg"]
    if msg["type"] == "ACTION":
        print(msg["tick"], msg["sender"], msg["action_type"], msg["action_class"])

# 예: 가드레일 별 block 사유 카운트
from collections import Counter
counter = Counter()
for ev in read_jsonl(f"runs/{ID}/guardrail.jsonl"):
    if ev["decision"] == "block":
        counter[(ev["guardrail_id"], ev["block_category"])] += 1
print(counter.most_common(10))
```

## 자주 묻는 분석 질문

### Q1. 이 run 에서 권한 위반이 몇 건 발생했나?

```python
r = json.load(open(f"runs/{ID}/report.json"))
print(len(r["violations"]["authority_violations"]))
```

### Q2. 어떤 가드레일이 가장 많이 차단했나?

```python
for gid, st in r["guardrail_summary"]["per_guardrail"].items():
    print(gid, st["decisions"].get("block", 0), "/", st["invocations"])
```

### Q3. 어택자가 어떤 채널을 가장 많이 사용했나?

```python
print(r["attack_summary"]["channel_distribution"])
```

### Q4. SCRAM 이 언제 발생했나?

```python
for ev in r["notable_events"]:
    if ev["kind"] == "SCRAM":
        print(ev)
```

### Q5. STA VETO 가 SRO OVERRIDE 로 뒤집힌 비율?

```python
v = r["agent_message_stats"]["sta_vetos"]
o = r["agent_message_stats"]["sro_overrides"]
print(f"override rate = {o}/{v}")
```

### Q6. 같은 시나리오 다른 가드레일 조합 비교

각 run 의 `report.json.configuration_summary.enabled_guardrails` 와 `attack_summary.successful_attack_count` 를 join 하면 ablation 매트릭스가 됩니다:

```python
import os, json
results = []
for run_id in os.listdir("runs"):
    p = f"runs/{run_id}/report.json"
    if not os.path.exists(p):
        continue
    r = json.load(open(p))
    results.append({
        "run_id": run_id,
        "guardrails": r["configuration_summary"]["enabled_guardrails"],
        "sta_mode": r["configuration_summary"]["sta_mode"],
        "scenario": r["configuration_summary"]["scenario"],
        "success": r["attack_summary"]["successful_attack_count"],
        "violations": len(r["violations"]["authority_violations"]),
        "csfs_lost": sum(1 for v in r["plant_trajectory"]["safety_function_final"].values() if v == "lost"),
    })
import pandas as pd
df = pd.DataFrame(results)
print(df)
```

## report 재생성

raw JSONL 만으로 `report.json` / `report.md` 재생성 가능:

```bash
python -m src.report --run-dir runs/<id>/
```

또는:
```bash
python -m src.report --run-id <id>      # ./runs/<id>/ 자동 해석
python -m src.report --run-dir runs/<id>/ --formats json,md
```

raw 가 canonical 이므로 report 가 손상되어도 언제든 재계산 가능. 스키마 업데이트 / 외부 분석 도구 추가 시에도 유용.

## 외부 분석 도구 연동

`manifest.json` 이 각 파일의 sha256 + line_count + schema_version 을 갖고 있어, 외부 도구는:

1. `manifest.json` 으로 무결성 검증
2. `report.json` 으로 헤드라인 통계
3. JSONL 들로 세밀한 분석

순으로 접근하면 됩니다.

JSONL 스키마는 본 문서 §"주요 JSONL 라인 스키마" 를 참조합니다.

## 무결성 검증

```bash
python -c "
import json, hashlib
m = json.load(open('runs/$ID/manifest.json'))
for art in m['artifacts']:
    p = f'runs/$ID/{art[\"path\"]}'
    h = hashlib.sha256(open(p, 'rb').read()).hexdigest()
    ok = h == art['sha256']
    print(art['path'], 'OK' if ok else 'CORRUPT')
"
```
