# 빠른 시작 — 5분 안에 첫 run 돌리기

## 사전 준비

- Python 3.11 이상 (또는 Docker 24+)
- (선택) LLM API 키 하나 이상 — 실제 LLM 으로 돌릴 때만 필요. mock 모드는 키 불필요.

## 옵션 A — 로컬 + mock LLM (가장 빠름, 키 불필요)

```bash
cd nuclear-redteam-sim
make install              # 가상환경 + 의존성 설치
make smoke                # 50 tick 동안 mock 운전팀이 normal_baseline 시나리오 돌림
```

진행되는 동안 콘솔에 `orchestrator_start`, `agent.SRO`, `guardrail` 등의 로그가 출력됩니다.

run 종료 후:
```bash
ls runs/                  # 새 run_id 디렉토리 확인
cat runs/<id>/report.md   # 사람-가독 리포트
```

`report.md` 의 §1 요약에서 가장 먼저 확인할 것:
- **결과 한 줄**: 안전기능 손상 개수 / 권한위반 건수 / 어택 세션 수
- **종료 사유**: `max_ticks` 가 정상

## 옵션 B — Docker 로 실제 LLM 사용

```bash
cp .env.example .env
# .env 편집: ANTHROPIC_API_KEY=sk-... 등 키 입력
make build                # 두 이미지 빌드 (simulator + llm_proxy)
make up                   # docker compose up -d
make logs                 # tail -f simulator 컨테이너
```

`make logs` 가 운전팀의 LLM 호출 로그 + 매 tick 상태 변화를 보여줍니다.

종료:
```bash
make down
```

`runs/<id>/` 는 호스트에 mount 되어 컨테이너 종료 후에도 남습니다.

## 첫 run 결과 빠르게 검토

```bash
ID=$(ls -t runs/ | head -1)
cat runs/$ID/report.md | head -40
```

`report.md` 의 핵심 섹션:
- §1 요약 — 한 줄 결과
- §3 가드레일 활동 — 각 G_i 가 몇 번 발동했고 차단/허용 비율
- §4 Plant Trajectory — 6 CSF (Critical Safety Functions) 최종 상태
- §5 권한 위반 / Bypass 시도 — 의미 있는 정책 위반 이벤트
- §6 Notable Events — SCRAM / EOP 진입 / CSF 손상 등 핵심 사건
